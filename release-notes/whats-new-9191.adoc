= What's new in ONTAP 9.19.1 for ASA r2 systems
:hardbreaks:
:nofooter:
:icons: font

[.lead]
Learn about the new capabilities available in ONTAP 9.19.1 for ASA r2 systems.

== Data protection
[cols="2,4" options="header"]
|===
| Update
| Description
| link:../data-protection/prepare-aix-snapmirror-active-sync.html[AIX host support for SnapMirror active sync]
| Transparent Application Failover for AIX is supported with SnapMirror active sync in symmetric active/active configurations.
|===

== Performance
[cols="2,4" options="header"]
|===
| Update
| Description
| link:../learn-more/additional-supported-ontap-features.html#performance[TCP performance improvements]
| ONTAP improves TCP data transfer performance over lossy, high-latency WAN links by integrating two loss-recovery features, PRR and RACK, into the ONTAP TCP stack. 
|===

| Description
| link:../install-setup/set-up-data-access.html#migrate-data-from-a-third-party-storage-system[ONTAP Foreign LUN Import (FLI) support for iSCSI protocol]

== SAN configuration

[cols="2,4" options="header"]
|===
| Update
| Description
| link:../learn-more/additional-supported-ontap-features.html#san-direct-attachment[Support for direct-attachment FC configurations]
| An FC switch is not required to establish connections between FC and FC-NVMe hosts and ONTAP storage.  Hosts can be directly connected to FC adapter ports on ASA r2 systems.  Direct-attachment capability can be especially valuable in remote sites and hybrid environments where simplicity is key. 
|===

== Security
[cols="2,4" options="header"]
|===
| Update
| Description

| link:../secure-data/enable-anti-ransomware-protection.html[Support for ARP/AI with SnapMirror active sync relationships]
| Autonomous Ransomware Protection with Artificial Intelligence (ARP/AI) supports primary read-write storage units in SnapMirror active sync relationships on ASA r2 systems. ARP/AI runs on the active primary storage units, creates primary-only ARP snapshots, and provides machine-learning-based ransomware detection for those storage units.

Clusters upgraded to ONTAP 9.19.1 that previously had default ARP enablement suppressed due to synchronous replication relationships will have it automatically restored. A pre-update warning notifies administrators and provides the option to disable it.

| link:../learn-more/additional-supported-ontap-features.html#tls-hardware-offload[Support for TLS hardware offload]
| You can offload Transport Layer Security (TLS) encryption and decryption for your entire cluster to supported Ethernet cards.  This reduces CPU overhead and improves performance for systems using TLS.

|===

== Storage efficiency
[cols="2,4" options="header"]
|===
| Update
| Description

| link:../learn-more/additional-supported-ontap-features.html#global-deduplication[Support for global deduplication]
| Global deduplication extends the existing ONTAP deduplication space savings to the entire Storage Availability Zone (SAZ). By eliminating duplicate data blocks across all the storage units in the SAZ, it can significantly improve storage efficiency and reduce total cost of ownership. Global deduplication is enabled by default and operates transparently without the need for administrative configuration. 

|===

== Storage resource management
[cols="2,4" options="header"]
|===
| Update
| Description

| link:../monitor/monitor-storage.html#monitor-storage-availability-zone-utilization[Support for storage availability zone capacity alert in System Manager]
| An alert is displayed in System Manager if a storage availability zone in your ASA r2 system nears capacity.

| link:../learn-more/additional-supported-ontap-features.html#modify-default-settings-that-control-automatic-qos-throughput-ceiling-increases[Support for modifying default settings that control automatic QoS throughput ceiling increases]
| You can modify the default settings that control how QoS policy groups automatically increase the QoS throughput ceiling in response to short-term needs for higher performance. These settings cannot be modified in ONTAP 9.18.1 and earlier.
|===

= Disable default autonomous ransomware protection on your ASA r2 storage systems
:icons: font
:imagesdir: ../media/

[.lead]
When you initialize a new 9.18.1 ASA r2 cluster or upgrade your cluster to 9.18.1, ARP/AI is automatically enabled by default on all new and existing storage units after a 12-hour grace period. If you do not disable ARP/AI during the grace period, it is enabled cluster wide when the grace period ends. 

.Steps

You can disable the default enablement during or after the initial 12-hour period.

[role="tabbed-block"]
====

.System Manager
--
. Select *Cluster>Settings*.
. Disable ARP:
* Disable during the 12-hour grace period:
.. Under *Anti-ransomware* select, *Don't enable* and then select *Disable*.

* To disable after the 12-hour grace period, 
.. Under *Anti-ransomware*, select image:icon_pencil.gif[Edit] and then deselect *Enable*.
.. Select *Save*
--

.Cli
--
. Check the default enablement status:
+
[source,cli]
----
security anti-ransomware auto-enable show
----

. Disable default enablement for existing and new volumes:
+
[source,cli]
----
security anti-ransomware auto-enable modify -default-existing-volume-state false -default-new-volume-state false
----
--

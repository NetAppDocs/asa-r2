= Cable the hardware for your ASA r2 storage system
:icons: font
:imagesdir: ../media/

[.lead]
Cable your ASA r2 storage system to enable cluster communication, host network access, management connectivity, and storage shelf connections. 

== Overview of hardware cabling

Cabling differs by platform model because each ASA r2 model has unique slot locations, port assignments, and I/O module configurations. Select the procedure for your storage system:

* link:cable-hardware-a1k.html[Cable ASA A1K]
* link:cable-hardware-a70-90.html[Cable ASA A70 and A90]
* link:cable-hardware-a20-30-50.html[Cable ASA A20, A30, and A50]
* link:cable-hardware-c30.html[Cable ASA C30]

You'll cable four types of connections during hardware cabling:

* *Cluster/HA connections*
+
Connect the two controllers to each other (switchless cluster) or to cluster network switches (switched cluster). These connections enable cluster communication and high-availability failover between controller pairs. Cluster interconnect traffic and HA traffic share the same physical ports.

* *Host network connections*
+
Connect the controllers to SAN hosts using Ethernet (iSCSI, NVMe/TCP) or Fibre Channel (FC, NVMe/FC) based on your host configuration. These connections provide data access for your storage workloads.

* *Management network connections*
+
Connect the controllers to your management network for system administration access through ONTAP System Manager and CLI.

* *Storage shelf connections*
+
Connect the controllers to NS224 storage shelves to provide storage capacity for your system.

== Additional resources

The cabling procedures show common configurations. The specific cabling depends on the components ordered for your storage system. For comprehensive configuration and slot priority details, see link:https://hwu.netapp.com[NetApp Hardware Universe^].
